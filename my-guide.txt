# 🌐 5. Networking Concept (VERY IMPORTANT)

By default:

* Pod is **NOT accessible outside**

So you need a Service:

### Types:

* ClusterIP → internal only
* NodePort → access via localhost (for you) ✅
* LoadBalancer → public (cloud)

👉 Use **NodePort** for now

# 🧪 8. How Your App Will Run in K8s

Flow:

1. Kubernetes pulls image from Docker Hub
2. Creates Pod
3. Injects env vars from Secret/ConfigMap
4. App starts
5. Service exposes it
